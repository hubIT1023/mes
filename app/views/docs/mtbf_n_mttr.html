In this system, reliability metrics are calculated using maintenance log data where:

  • col_10 = 'MAINT-COR' marks a corrective maintenance event (used as failure occurrence)
  • col_3 = 'PROD' indicates the system has returned to production (repair completion)

All timestamps come from col_6.

‚è± MTBF ‚Äì Mean Time Between Failures

Definition: Average operational time between consecutive corrective maintenance events.

Purpose: Measures reliability. Higher MTBF = more reliable asset.

MTBF = Total Time Between Consecutive MAINT-COR Events √∑ Number of Intervals

How It's Calculated

  1. Select all rows where col_10 = 'MAINT-COR'
  2. For each asset, sort events by timestamp (col_6)
  3. Use LEAD() to get the next MAINT-COR timestamp
  4. Compute time difference in hours
  5. Average all intervals per asset

üõ† MTTR ‚Äì Mean Time To Repair

Definition: Average time from a corrective maintenance event to system restoration.

Purpose: Measures maintainability. Lower MTTR = faster recovery.

MTTR = Total Downtime Across All Repairs √∑ Number of Repairs

How It's Calculated

  1. For each MAINT-COR event, treat its timestamp as failure start
  2. Find the earliest subsequent col_3 = 'PROD' event for the same asset
  3. Compute time difference in hours
  4. Average all valid repair durations per asset

Definition: Percentage of time a system is operational and available for use.

Purpose: Holistic view of reliability + maintainability.

Availability (%) = [ MTBF √∑ (MTBF + MTTR) ] √ó 100%

How It's Calculated

  1. Compute MTBF and MTTR for the same asset(s)
  2. Apply the formula above
  3. Result is expressed as a percentage (0‚Äì100%)

Example:
